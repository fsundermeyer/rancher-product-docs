= 集群自动扩缩器
ifeval::["{build-type}" != "srfa"]
:page-languages: [en, zh]
endif::[]
:revdate: 2025-08-05
:page-revdate: {revdate}

在本节中，您将学习如何在使用 AWS EC2 自动扩展组的 Rancher 自定义集群上安装和使用 https://github.com/kubernetes/autoscaler/blob/master/cluster-autoscaler/[Kubernetes cluster-autoscaler]。

集群自动扩缩器是一个工具，当以下条件之一为真时，它会自动调整 Kubernetes 集群的大小：

* 由于资源不足，集群中有无法运行的Pod。
* 集群中有节点在长时间内未被充分利用，其Pod可以放置在其他现有节点上。

为了防止您的 pod 被驱逐，请在 pod 规范上设置 `priorityClassName: system-cluster-critical` 属性。

集群自动扩缩器旨在运行在 Kubernetes 主节点上。它可以在 `kube-system` 名称空间中运行。集群自动扩缩器不会缩减运行非镜像 `kube-system` pod 的节点。

可以在工作节点上运行集群自动扩缩器的自定义部署，但需要额外小心以确保集群自动扩缩器保持运行。
